# Context: resolved domain glossary

The vocabulary used throughout the Writing Engine code and docs. This file is the
single source of truth for what each term means in this project.

| Term                                    | Meaning in this project                                                                                                                                                                                     |
| --------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Heartbeat**                           | The time/state-driven loop that wakes the agent on a tick, checks the source, and runs a cycle — as opposed to being triggered by a human prompt. Implemented by `Heartbeat` in `src/core/heartbeat.ts`.    |
| **Cycle**                               | One full pass through the pipeline: poll → snapshot → research → write → validate → evaluate → extract lessons → integrate memory → persist. Implemented by `runCycle` in `src/core/pipeline.ts`.           |
| **Source event**                        | A single item from a live public feed (`SourceEvent`). In the demo it comes from a deterministic fixture.                                                                                                   |
| **Snapshot / provenance**               | An immutable, content-hashed capture of a source event at retrieval time (`SourceSnapshot`). The snapshot is the unit of provenance; "live source is truth."                                                |
| **Evidence pack**                       | The researcher's structured output: claims (each with a supporting URL), novelty (what changed vs. prior state), uncertainties, and metric deltas (`EvidencePack`).                                         |
| **Artifact**                            | A generated piece of writing plus how it was produced (`Artifact`). The scaffold's single format is a **decision memo**.                                                                                    |
| **Writing task**                        | The audience/format/length/citation contract an artifact must satisfy (`WritingTask`).                                                                                                                      |
| **Deterministic validator**             | A model-free, objective check run before any rubric judgement (citations present, not stale, no injection markers, required sections, etc.). Produces `ValidatorResult`s.                                   |
| **Rubric**                              | The frozen set of seven scored dimensions: source fidelity, insight, audience usefulness, structure, style, freshness, safety. Versioned (`rubric@1`).                                                      |
| **Rubric evaluator**                    | The independent scorer that rates each dimension in [0, 1] (`Evaluation`). Separate from the writer; may **abstain** on failure rather than emit a fake zero.                                               |
| **Aggregate**                           | The mean score across the seven rubric dimensions for one artifact; `null` when the evaluator abstained.                                                                                                    |
| **Abstention**                          | An evaluator's explicit "no trustworthy judgement" result. Scores are `null` and are never coerced to zero; the engine learns nothing from an abstained cycle.                                              |
| **Lesson**                              | A validated, reusable rule extracted from critique (`Lesson`). Carries a human-readable `rule`, a machine-applicable `directive`, a `targetDimension`, `scope`, evidence, confidence, and win count.        |
| **Directive**                           | The machine-applicable form of a lesson (`LessonDirective`) that the writer knows how to apply. In the demo it toggles a concrete change in the memo; in production it maps to a prompt injection.          |
| **Episodic lesson**                     | A newly learned lesson that has not yet earned enough wins to be promoted.                                                                                                                                  |
| **Promotion / playbook**                | A lesson is promoted to the durable playbook after repeated wins (`PROMOTION_THRESHOLD`). The playbook is the compounding, durable knowledge.                                                               |
| **Win / reinforcement**                 | An applied lesson whose target check passes on a later cycle. Wins raise confidence and drive promotion.                                                                                                    |
| **Scope**                               | The tag (feed/domain) that limits where a lesson applies and is retrieved. Lessons generalize within a scope.                                                                                               |
| **Held-out task**                       | A benchmark task the engine never learns from. Its improvement can only come from lessons learned on other tasks — a guard against overfitting.                                                             |
| **Baseline → latest delta**             | The core recursive-intelligence metric: the difference between the first (lesson-free) evaluation and the most recent one, per dimension and aggregate.                                                     |
| **Port**                                | An interface in `src/ports/` representing a replaceable seam (source, researcher, writer, validator, evaluator, extractor, store).                                                                          |
| **Adapter**                             | A concrete implementation of a port. The scaffold's adapters are deterministic and offline.                                                                                                                 |
| **Demo heuristic vs. production model** | The demo's researcher/writer/evaluator are deterministic heuristics chosen for reproducibility and zero-dependency running. The production versions (Nemotron/vLLM-served models) implement the same ports. |
| **Store**                               | Persistent memory + run history (`Store`). The scaffold uses versioned filesystem JSON; Supabase is a defined-but-unimplemented port.                                                                       |
| **Human publishing gate**               | The rule that nothing is published without explicit human approval. There is no auto-publish path anywhere in the code.                                                                                     |
